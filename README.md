二、核心技术知识点，贴合自然资源场景
 
1. RAG检索增强（核心需求）
 
1. 完整流程：文档加载→文本分割（适配地质报告/林业规范长文本）→向量化Embedding→向量数据库存储（Chroma/Pinecone/Milvus）→多路检索（语义+关键词重排）→上下文拼接进Prompt→LLM生成回答
​
2. 落地优化点
 
- 行业文档适配：PDF图纸、勘察表格、政策法规混合解析
​
- 精度优化：分层切片、父文档检索、重排序Rerank、过滤无关文档
​
- 解决痛点：大模型行业知识缺失、回答幻觉、无法调取内部档案
 
2. Agent智能工作流
 
1. 核心能力：自主判断任务、调用自定义工具、多步骤拆解、多轮迭代完成复杂业务需求
​
2. 对应公司业务工具示例（贴合林业/地质）
 
- 文档查询工具：调取矿山修复、林地勘测历史资料
​
- 测算工具：碳汇计算、边坡风险指标计算
​
- 生成工具：自动输出地质评估报告、林业实施方案
 
3. LangChain实现方案：ReAct Agent、Plan-and-Solve规划Agent、工具调用解析器
 
3. LangChain vs LlamaIndex 分工（必知）
 
- LlamaIndex：强项是文档索引、知识库、RAG检索链路，擅长海量专业文档的存储与检索，适合搭建企业私有知识库；
​
- LangChain：强项是流程编排、Agent工具链、LLM对话链路、多组件串联，适合复杂业务流程、多工具智能体开发；
​
- 工程落地组合方案：LlamaIndex做底层向量检索底座，LangChain封装上层Agent业务逻辑，二者协同搭建完整AI解决方案。
 
三、高频问答（该岗位定向）
 
Q1：如果让你做一套林业地质档案智能问答系统，你的RAG整体设计思路？
答：
 
1. 数据源处理：批量解析PDF勘察报告、林地政策、矿山修复标准，统一清洗、分块；
​
2. 索引构建：LlamaIndex构建分层向量索引，启用父文档检索提升长文本完整度；
​
3. 检索优化：语义检索+关键词混合召回，增加行业术语重排序模型过滤低相关片段；
​
4. 上层Agent：LangChain封装检索工具、数据计算工具，复杂问题自动分步检索、汇总生成专业答复；
​
5. 工程部署：Docker打包服务，K8s实现多实例并发，输出POC演示系统交付业务部门验证。
 
Q2：Agent工作流怎么解决自然资源复杂方案生成需求？
答：使用Plan-and-Solve Agent，先拆解用户需求（如矿山生态修复方案），自主判断需要调取的规范文件、测算数据，依次调用检索、计算工具，整合全部信息后生成完整方案；
同时增加输出校验工具，对生成内容的合规性、数据准确性做二次校验，降低大模型幻觉。
